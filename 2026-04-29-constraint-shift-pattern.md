# The Constraint-Shift Pattern

## The Idea

A universal pattern observed across technology, biology, energy, and economics:

> **Solutions don't eliminate constraints — they shift bottlenecks to the next layer.**

Every system that solves one constraint reveals the constraint beneath it. The bottleneck doesn't disappear; it relocates.

---

## The Pattern

```
System faces Constraint A
      ↓
Solution developed for Constraint A
      ↓
Constraint A resolved
      ↓
Constraint B (previously hidden) becomes the new bottleneck
      ↓
Repeat
```

The level of abstraction rises with each cycle. The new constraint is typically at a *lower* layer of the stack — closer to physical limits or political control.

---

## Evidence Across Domains

### AI Infrastructure
- **Constraint A:** Closed-source AI models controlled access
- **Solution:** Open source (Meta Llama, Mistral) — software layer freed
- **Constraint B:** HBM memory (Samsung/SK Hynix monopoly) — hardware layer constrained
- **Predicted next:** Energy supply as compute scales — physical layer

### Genomics
- **Constraint A:** Sequencer cost and size ($10M+ machines, lab-only)
- **Solution:** Oxford Nanopore MinION — £99, pocket-sized
- **Constraint B:** Compute (basecalling too slow on consumer hardware)
- **Solution:** Apple M4 silicon — 23x faster basecalling
- **Constraint C:** Consumables (flow cells £500-800 each) — now the bottleneck

### Battery Technology
- **Constraint A:** Battery cell design (energy density, charging speed)
- **Solution:** Tesla dry electrode manufacturing — design constraint solved
- **Constraint B:** Lithium input supply — raw material constraint emerges

### Cryptocurrency
- **Constraint A:** Centralised financial rails controlled value transfer
- **Solution:** Bitcoin/crypto — decentralised value layer
- **Constraint B:** Regulatory capture — institutions re-centralise at protocol layer
- **Predicted next:** Physical infrastructure (mining, nodes) as new control point

### Biology
- **Constraint A:** Gene editing precision — random mutation, no targeting
- **Solution:** CRISPR — precise gene editing achieved
- **Constraint B:** Delivery mechanism — getting CRISPR into the right cells

## Predictive Value

The pattern is useful for:

1. **Identifying the next investment bottleneck** — once a layer is solved, look one layer down
2. **Avoiding over-optimism** — "problem solved" almost never means "problem gone"
3. **Finding durable moats** — whoever controls the next constraint layer captures disproportionate value
4. **Technology timing** — the constraint-shift moment is when the previous layer commoditises and new infrastructure value concentrates

---

## Related Framework

Connects to the "Infrastructure Layer = Real Moat" thesis: as application layers commoditise, infrastructure layers (which become the new constraints) retain pricing power.
